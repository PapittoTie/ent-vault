## Система обучения
Obsidian vault — база знаний с атомарными заметками. Одна заметка = одна концепция.
Процесс: читаю заметку → закрываю → пишу интуицию своими словами → удаляю тег #статус/не_изучена.

## Формат заметки (строго по ENT_TEMPLATE.md)

1. Теги вверху: `#предмет/раздел #статус/не_изучена`
2. `## Интуиция` — ВСЕГДА ПУСТАЯ. Пользователь заполняет сам после чтения. Не писать сюда ничего.
3. `## Определение` — формальное, ключевые термины жирным, [[ссылки]] на существующие заметки
4. `## Примеры` — конкретные примеры + визуализация (YouTube или GIF, только эти два)
5. `## Контекст` — связи через Extended Graph синтаксис (см. ниже). Только на существующие заметки.
6. `## Упражнение` — 2 вопроса, каждый подписан что именно проверяет
7. `## Источники` — минимум 2, только официальные или достоверные

## Как создавать заметку (процесс)
Перед написанием заметки — обязательный ресёрч:
1. Найти минимум 10 источников по теме (поиск в интернете)
2. Прочитать и сравнить трактовки
3. Только после этого писать заметку — своими словами, не копировать
Цель: заметка должна отражать полный контекст темы, а не один учебник.

## Extended Graph — синтаксис связей
Установлены плагины: **Extended Graph** + **Dataview**.

Типы связей указываются в **YAML frontmatter** каждой заметки:

```yaml
---
tags:
  - предмет/раздел
  - статус/не_изучена
причина:
  - "[[Заметка]]"
следствие:
  - "[[Заметка]]"
путают:
  - "[[Заметка]]"
---
```

Плагин окрашивает рёбра графа по типу. Цвета:
Settings → Extended Graph → Link types → Specific colors:
`путают` → красный, `следствие` → синий, `причина` → жёлтый.

Секция `## Контекст` в заметках УДАЛЕНА — связи теперь только в frontmatter.

## Правила
- Заголовок заметки = имя файла. Внутри заметки заголовка нет.
- Тег #статус/не_изучена убирается пользователем когда написана интуиция
- [[Ссылки]] использовать везде в тексте, не только в Контексте
- В Контексте — только Extended Graph синтаксис, не markdown списки
